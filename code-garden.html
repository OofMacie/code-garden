HTML notes
🪴
HTML gives a page its structure — think of it like the skeleton. It tells the browser what things are (a heading, a paragraph, a link), not how they look. CSS does the styling later.
🧱 basic document structure
today
Every HTML file starts the same way. The <!DOCTYPE html> declaration tells the browser we're using HTML5. Then comes <html> wrapping everything, with <head> (invisible metadata) and <body> (visible content).
<!-- every HTML file needs this ✓ --> <!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8"> <title>my page</title> </head> <body> <!-- all visible content goes here --> </body> </html>
structure boilerplate ch.1
📝 common tags to know
this week
Semantic tags mean something — they describe the content's role. Non-semantic tags like <div> and <span> are just containers with no meaning attached.
<!-- headings: h1 (biggest) → h6 --> <h1>main title</h1> <h2>section title</h2> <!-- text --> <p>a paragraph of text</p> <strong>bold / important</strong> <em>italic / emphasis</em> <!-- links & images --> <a href="url">click me</a> <img src="photo.jpg" alt="desc">
tagssemanticch.2
📦 lists & containers
this week
Lists come in two flavors. Unordered (<ul>) gives bullet points. Ordered (<ol>) gives numbers. Both use <li> for each item. <div> and <section> group other elements.
<ul> <!-- unordered (bullets) --> <li>apples</li> <li>oranges</li> </ul> <ol> <!-- ordered (numbers) --> <li>first</li> <li>second</li> </ol>

Python basics — pre-notes
🐍 Python variables & types
reference
Python is dynamically typed — you don't declare types, Python figures them out. Everything is an object. Use type() to check what type something is.
# no type declarations needed! name = "macie" # str age = 22 # int score = 9.8 # float is_artist = True # bool print(type(name)) # <class 'str'>

🔄 functions in Python
reference
Functions use def and indentation (4 spaces) instead of curly braces. Python cares a lot about whitespace — it's part of the syntax.
def greet(name): return f"hello, {name}! 🌿" def add(a, b): return a + b print(greet("macie")) # hello, macie! 🌿 print(add(3, 4)) # 7

CS concepts to know
Big-O Notation — why it matters

Big-O describes how an algorithm's speed scales with input size. O(1) is constant (always same speed). O(n) grows linearly with input. O(n²) gets slow fast. The goal is usually to minimize Big-O, especially for large data sets.

HTTP Methods — GET, POST, PUT, DELETE

GET retrieves data (safe, no changes). POST creates new data. PUT updates existing data. DELETE removes it. These are the building blocks of any REST API — every web request uses one of these.

Variables vs. Functions — the mental model

A variable stores a value (data at rest). A function stores a procedure (actions to run). Think of variables as nouns and functions as verbs. In Python, functions are also objects — you can pass them around like values.

🎯 CSS flexbox centering
The one trick you'll use everywhere — centering literally anything.
.center { display: flex; align-items: center; justify-content: center; }

📐 CSS box model reset
Put this at the top of every CSS file — makes sizing predictable.
* { margin: 0; padding: 0; box-sizing: border-box; }
CSSreset
🌐 HTML link template
target="_blank" opens in new tab. Always add rel for security.
<a href="https://example.com" target="_blank" rel="noopener noreferrer" > link text here </a>

🎨 CSS custom properties
Define colors once at the top, use everywhere — easier to update later.
:root { --sage: #7BAE7F; --cream: #FDFAF5; --text: #3A3228; } body { background: var(--cream); color: var(--text); }
